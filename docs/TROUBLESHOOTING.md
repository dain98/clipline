# Diagnostics and troubleshooting

Open **Settings > Support** to see the active diagnostics directory. You can
also right-click the Clipline tray icon and choose **Open Diagnostics Folder**;
that native action remains available if the app UI fails to load.

When reporting a problem:

1. Reproduce the failure if it is safe to do so.
2. Enter a 10–4,000 character description with what you did, what you expected,
   what happened, and roughly when.
3. Choose **Prepare Report** and review the exact file list and compressed size.
4. Choose **Send Private Report** only if you approve the disclosed contents.

If offline or upload fails, choose **Save a Copy**. Prepared reports expire
after 30 minutes and abandoned staging data is pruned; Clipline never uploads a
saved or failed report later by itself.

The success screen provides a report ID. It is not a public tracking link—it is
useful only when communicating directly with the Clipline developer. Reports
are private and expire after 30 days.
